## Design Context

### Users
**混合用户群体**，覆盖不同技术层次的人群：
- AI 从业者/研究者 — 深度验证 AI 回答质量，关注准确性
- 普通 AI 使用者 — 日常使用 AI 但希望交叉验证答案
- 学生/学习者 — 用 AI 辅助学习，通过社区判断答案正确性

**使用场景**：用户在 AI 对话中获得答案后，分享到社区供他人审视、讨论、验证。这是一个"AI 答案的检验场"，核心行为是提问 → 判断 → 分享 → 讨论。

### Brand Personality
**品牌三词**：深邃 · 通透 · 社区感

- 像深夜实验室的暖光——**专注、沉静、有探索氛围**
- **不冷、不炫、不跟风**——深色不等于赛博朋克霓虹风
- 鼓励从私有用到公共讨论的**自然过渡**——从"我得到了答案"到"我们一起来验证"

### Aesthetic Direction

**视觉调性**：深夜实验室 × 紫调暗色社群空间

**主题**：深色主题（纯暗色系），葡萄紫作为品牌主色

**主色调**：葡萄紫（OKLCH hue ≈ 280–290°）——代表沉思、探索、知识深度。紫色同时混合了蓝色的理性与红色的热情，适合一个"深度思考+社区讨论"的产品。

**反参考**：
- ❌ 赛博朋克霓虹紫——不要过量发光/辉光效果
- ❌ 企业级后台/管理面板——避免密集表格/冰冷灰
- ❌ 典型的 AI chat UI（白底蓝按钮）

**情绪目标**：让人感到沉静、专注、安全——像一个深夜还在亮着灯的实验室，几个人围在一起讨论一个答案的对错。

### Design Principles

1. **紫调暗色，沉稳深邃** — 葡萄紫作为品牌主色，深色背景为主体。紫色选择偏灰紫/矿物紫而非亮紫/霓虹紫，保持高级感。

2. **光从文字来** — 界面本身是暗的，但内容（AI 回答、用户讨论）是明亮的。文字是光源，UI 只是容器。这种对比让注意力自然聚焦到内容上。

3. **圆润亲和，不冷** — 深色 UI 容易显得冷冰冰。通过温润的圆角（比常规大25%）、暖紫调的中性色、克制的微光效来保持亲和力。

4. **内容为王，对话优先** — AI 回答和社区讨论是页面的主角。界面不该喧宾夺主，深色背景本身就是减法设计。

5. **渐进式参与** — 从私有对话（安静、专注）→ 分享弹窗（微动效升温）→ 广场（社区活力），每个阶段视觉情绪逐层递进。

6. **避免管理面板感** — 不搞密集数据表格、树形导航、灰底卡片。保持宽松间距、可读排版、人性化文案。

### Technical Constraints
- React 19 + Vite + Tailwind CSS 3
- 深色主题（`class` 模式 dark mode，但默认为 dark）
- 所有颜色使用 OKLCH
- 中文内容为主，需要良好的中文字体渲染
